**버전**: 0.1 (설계 합의용 초안)
**작성일**: 2026-06-18
**원칙**: Doc-First (이 문서 확정 후 코드) · Test-First (모듈마다 pytest 100% 후 적용)

**결과물**: 일일 발굴 랭킹/알림 + **백테스트 검증 루프** (적중률 증명 후 사용)

## 제0부: 철학 — "폭등 임박"이란 무엇인가

### 0.1 우리가 잡으려는 것 / 잡지 않으려는 것

| 잡는다 (✅) | 안 잡는다 (❌) |
|---|---|
| 압축이 끝나 **터지기 직전**인 종목 | 이미 +30% 터져버린 종목 추격 |
| 거래량 마르고 변동성 수축된 **바닥/베이스** | 고점에서 거래량 터지며 분배 중인 종목 |
| 지수 대비 강한 **주도주 후보** | 지수 따라 흐느적대는 약세 종목 |
| 수급(외국인·기관)이 **조용히 매집**한 종목 | 개인만 들어와 있고 기관이 던지는 종목 |

### 0.2 인정하는 한계 (정직하게)

- **폭등을 100% 예측하는 것은 불가능하다.** 우리가 하는 건 "무작위로 찍을 때보다 적중률을 몇 배(lift) 높이는" 확률 게임이다.
- 따라서 **검증되지 않은 점수는 쓰지 않는다.** 제5부 백테스트에서 "상위 점수 종목의 폭등 적중률이 시장 평균(base rate)보다 통계적으로 의미 있게 높다"가 증명되어야 실전 알림에 투입한다.
- 과최적화(overfitting)는 적이다. 파라미터는 적게, 규칙은 단순하게, 검증은 워크포워드(out-of-sample)로.

### 0.3 폭등의 조작적 정의 (백테스트 라벨 — 가장 중요한 한 줄)

> 신호일 `t`의 종가를 기준으로, **향후 K 거래일 내 고가가 +X% 이상** 오르면 그 신호를 "폭등 적중(label=1)"으로 본다.

| 트랙 | K (거래일) | X (상승률) | 의미 |
|---|---|---|---|
| **단기** | 10 | +20% | 2주 내 돌파 점화 |
| **중기** | 30 | +40% | 6주 내 본격 시세 |

> 이 4개 숫자(K·X × 2트랙)는 **설계의 핵심 손잡이**다. 기본값으로 시작하되 제5부에서 민감도 분석으로 조정한다. 합의 시 여기서 확정.

## 제1부: 시스템 개요 (파이프라인)

```
┌──────────────────────────────────────────────────────────────────┐
│  [1] 데이터 수급 (datafeed.py)                                     │
│  FDR/pykrx → 전종목 일봉 OHLCV + 수급 + 시총 → 로컬 캐시(parquet) │
└───────────────┬──────────────────────────────────────────────────┘
                │
┌───────────────▼──────────────────────────────────────────────────┐
│  [2] 유니버스 필터 (universe.py)                                   │
│  관리/경고/스팩/우선주/ETF 제외 + 최소 거래대금·시총 게이트       │
└───────────────┬──────────────────────────────────────────────────┘
                │  (살아남은 깨끗한 종목만)
┌───────────────▼──────────────────────────────────────────────────┐
│  [3] 팩터 계산 (factors.py)  ※ 포인트인타임, 룩어헤드 금지        │
│  단기 F1~F4 (수축/거래량/박스/점화) + 중기 F5~F8 (베이스/RS/추세/수급)│
└───────────────┬──────────────────────────────────────────────────┘
                │
┌───────────────▼──────────────────────────────────────────────────┐
│  [4] 스코어링 (score.py)                                          │
│  필수 게이트 통과 → 횡단면 백분위 가중합 → 단기/중기/종합 0~100   │
└──────┬──────────────────────────────────────┬────────────────────┘
       │                                       │
┌──────▼─────────────────┐      ┌──────────────▼────────────────────┐
│ [5a] 일일 스캔(scanner)│      │ [5b] 백테스트 검증(backtest.py)   │
│ 랭킹 → 리포트/텔레그램 │      │ 라벨링 → Precision/Lift/MFE 측정  │
└────────────────────────┘      └───────────────────────────────────┘
```

**핵심 분리**: `factors.py`/`score.py`는 **"오늘까지의 데이터로 오늘의 점수를 낸다"** 는 동일 함수를 일일 스캔과 백테스트가 **똑같이** 호출한다. 백테스트와 실전이 같은 코드를 쓰므로 "백테스트는 좋았는데 실전은 다르다"는 괴리가 원천 차단된다.

## 제2부: 데이터 계층 (datafeed.py)

### 2.1 수급 소스 매핑

| 데이터 | 소스 | 함수(예시) | 비고 |
|---|---|---|---|
| 전종목 코드/이름 | pykrx | `stock.get_market_ticker_list(date, market)` | KOSPI/KOSDAQ |
| 일봉 OHLCV | FDR | `fdr.DataReader(code, start, end)` | 빠르고 안정적 |
| 거래대금 | OHLCV 파생 | `close * volume` | 유동성 게이트용 |
| 시가총액 | pykrx | `stock.get_market_cap(date)` | 소형주 필터 |
| 외국인/기관 순매수 | pykrx | `stock.get_market_trading_value_by_date(...)` | 수급 팩터 F8 |
| 관리/경고 지정 | pykrx/KRX | `get_market_ohlcv` 부가 or 별도 리스트 | 유니버스 필터 |

> 수급(외국인/기관)은 종목별 일별 호출이라 느리다 → **상위 후보(유니버스 통과분)에만** 2차로 조회하거나, 전종목 일괄 API를 캐시. 1차 구현에서는 F8을 "있으면 가점"인 선택 팩터로 두고, 가격/거래량 팩터로 후보를 좁힌 뒤 수급을 덧입힌다.

### 2.2 로컬 캐시 전략

```
data/
├── ohlcv/                  # 종목별 일봉 (parquet, 증분 업데이트)
│   ├── 005930.parquet
│   └── ...
├── universe/
│   ├── tickers_YYYYMMDD.parquet     # 그 날의 전종목 + 분류/시총
│   └── excluded_YYYYMMDD.parquet    # 관리/경고/스팩 등 제외 목록
└── supply/                 # 수급 (외국인/기관) — 후보 종목만
    └── 005930.parquet
```

- **증분 업데이트**: 매일 장 마감 후, 마지막 캐시일 다음날부터 오늘까지만 받아 append.
- **초기 적재**: 백테스트용으로 과거 N년(기본 5년) 1회 벌크 다운로드.
- 캐시 포맷은 parquet(빠름) 기본, 환경에 pyarrow 없으면 csv 폴백.

## 제3부: 유니버스 필터 (universe.py) — 쓰레기를 먼저 버린다

폭등 스캐너에서 **가장 과소평가되는 단계**. 노이즈·사기·유동성 함정을 먼저 거르지 않으면 점수가 아무리 좋아도 못 산다.

### 3.1 하드 제외 (무조건 탈락)

| 항목 | 이유 |
|---|---|
| 관리종목 / 투자위험·경고·주의 | 거래정지·상폐 리스크 |
| 거래정지 / 정리매매 | 매매 불가 |
| 우선주 (`*5`, `*7`, `*K` 등 코드 규칙) | 유동성·왜곡 |
| 스팩(SPAC) | 합병 이벤트 종목, 패턴 무의미 |
| ETF / ETN / 리츠(선택) | 개별 폭등 로직 대상 아님 |
| 신규 상장 N일 미만(기본 60일) | 데이터·변동성 불안정 |

### 3.2 소프트 게이트 (설정값, 기본값 제시)

```python
min_avg_value_krw   = 1_000_000_000   # 20일 평균 거래대금 ≥ 10억 (유동성)
min_market_cap_krw  = 30_000_000_000  # 시총 ≥ 300억 (초소형 작전주 회피)
min_price           = 1_000           # 동전주 제외 (호가 왜곡)
max_price           = None            # 상한 없음
exclude_limit_up_today = True         # 당일 상한가/급등(>+15%)은 추격 금지
```

> 게이트는 전부 `surge_config.py`로 노출 → 백테스트에서 민감도 분석 가능.

## 제4부: 폭등 임박 멀티팩터 모델 (factors.py) — 시스템의 심장

각 팩터는 **0.0~1.0 점수**를 반환한다 (1=강한 신호). 모든 계산은 **신호일까지의 캔들만** 사용한다(룩어헤드 금지).

### 4.A 단기 트랙 — "돌파 임박" (수일~2주)

#### F1. 변동성 수축 (VCP / 스퀴즈)
스프링이 압축된 상태. `indicators.bollinger_bands`의 밴드폭 활용.
```
bw[i] = (upper[i] - lower[i]) / middle[i]        # 밴드폭 비율(정규화)
score_F1 = 1 - percentile_rank(bw[-1], bw[-126:]) # 최근 6개월 중 현재 밴드폭이 하위면 高점
# 추가 조건: 최근 20봉 고가-저가 레인지가 단계적으로 축소(연속 수축)면 가점
```

#### F2. 거래량 마름 → 점화
바닥에서 거래량 고갈 후 첫 폭발(세력 진입 흔적).
```
vol_ma20 = SMA(volume, 20)
dryness  = 1 - percentile_rank(min(vol[-5:]), vol[-60:])  # 최근 거래량 바닥인가
ignition = clip(vol[-1] / vol_ma20[-1] / 3.0, 0, 1)        # 당일 거래량 ≥ 3배면 만점
score_F2 = 0.5*dryness + 0.5*ignition
```

#### F3. 박스 상단 / 신고가 근접 (피봇 임박)
돌파 직전 위치 에너지.
```
box_high = max(high[-60:-1])          # 최근 60일 박스 상단
dist     = (box_high - close[-1]) / box_high
score_F3 = clip(1 - dist/0.05, 0, 1)  # 박스 상단 -5% 이내면 만점, 멀수록 감점
# 이미 박스 상단을 +3% 넘게 돌파했으면(추격 영역) 감점 처리
```

#### F4. 단기 모멘텀 점화
```
rsi_now = rsi(close,14)[-1]
turn    = 1 if (rsi[-2] < 50 <= rsi[-1]) else 0.5 if rsi_now>50 else 0  # 50선 상향
above_ema20 = 1 if close[-1] > ema(close,20)[-1] else 0
score_F4 = 0.6*turn + 0.4*above_ema20
```

### 4.B 중기 트랙 — "베이스 빌드업" (수주~수개월)

#### F5. 베이스 성숙도
충분히 다진 토대일수록 돌파가 강하다.
```
base_len   = 횡보(가격이 ±15% 박스) 지속 거래일 수
maturity   = clip(base_len / 35, 0, 1)          # 7주(35일)+ 면 만점
contraction= 조정폭이 단계적으로 줄었는가(VCP 깊이 축소)  # 0~1
score_F5 = 0.6*maturity + 0.4*contraction
```

#### F6. 상대강도 (RS) — 지수 대비 초과수익
주도주의 1순위 조건.
```
ret_stock_60 = close[-1]/close[-60] - 1
ret_index_60 = kospi[-1]/kospi[-60] - 1          # 코스닥 종목은 코스닥지수
rs = ret_stock_60 - ret_index_60
score_F6 = percentile_rank(rs, 전종목 당일 rs 분포)  # 횡단면: 상위면 高
```

#### F7. 추세 토대 (장기 정렬)
```
above_ema200 = close[-1] > ema(close,200)[-1]
ema_stack    = ema60[-1] > ema120[-1] > ema200[-1]   # 정배열
near_52w_high= close[-1] / max(high[-252:]) > 0.75   # 52주 고가 75% 이상 (바닥 아님)
score_F7 = mean([above_ema200, ema_stack, near_52w_high])
```

#### F8. 수급 매집 (선택·가점) — 한국시장 특화
```
obv_div   = OBV는 상승하는데 가격은 횡보/하락 → 매집 다이버전스 (0~1)
inst_buy  = 외국인+기관 최근 20일 누적 순매수 > 0 그리고 증가추세 (0~1)
score_F8 = 0.5*obv_div + 0.5*inst_buy   # 수급 데이터 없으면 obv_div만
```

### 4.C 점수 합성 (score.py)

**2단계: 게이트(필수) → 랭크 가중합(상대)**

```
# 1) 필수 게이트 (하나라도 실패면 제외 — '말이 되는 후보'만 통과)
gate = (유니버스 통과)
       AND (close > ema200  OR  base 형성 중)       # 바닥 쓰레기 배제
       AND (거래대금 게이트 통과)

# 2) 트랙별 점수 (게이트 통과분만, 횡단면 정규화)
short_score = 100 * Σ w_short[i] * F[i]   # F1~F4, 가중치 합=1
mid_score   = 100 * Σ w_mid[i]   * F[i]   # F5~F8
total_score = max(short_score, mid_score) + 0.2*min(...)  # 한 트랙이라도 강하면 부각

# 3) 등급
S: ≥85 / A: ≥70 / B: ≥55 / C: <55 (C는 알림 제외)
```

> **가중치 초기값**(합의·튜닝 대상):
> 단기 `w = {F1:0.30, F2:0.30, F3:0.25, F4:0.15}`
> 중기 `w = {F5:0.25, F6:0.30, F7:0.25, F8:0.20}`
> 백테스트로 **각 팩터의 단독 lift를 측정**해 기여 없는 팩터는 가중치 0으로 쳐낸다.

## 제5부: 백테스트 검증 프레임워크 (backtest.py) — 사용자 핵심 요구

### 5.1 목적
"이 점수가 진짜 폭등을 **선행**했는가?"를 과거 데이터로 정량 증명한다. 통과 못 하면 실전 알림에 안 쓴다.

### 5.2 룩어헤드(미래참조) 차단 — 절대 원칙
```
for 평가일 t in 과거_거래일들:
    point_in_time_candles = candles[:t+1]      # t 까지만! t+1 이후 절대 금지
    score = score_engine(point_in_time_candles) # 실전과 동일 함수
    label = 1 if max(high[t+1 : t+1+K]) >= close[t]*(1+X) else 0  # 미래는 라벨에만
    기록.append((t, code, score, label, MFE, MAE))
```
- **점수 입력**과 **라벨 계산**의 시점 경계를 코드로 강제. 이를 깨면 백테스트가 거짓말을 한다 → 전용 단위테스트로 봉인(제8부).

### 5.3 평가 지표

| 지표 | 정의 | 통과 기준(예시) |
|---|---|---|
| **Base rate** | 전체 표본 중 폭등(label=1) 비율 | 기준선 |
| **Precision@등급S** | S등급 중 실제 폭등 비율 | — |
| **Lift** | Precision / Base rate | **≥ 2.0** (랜덤의 2배+) |
| **분위별 적중률** | 점수 10분위별 label 평균 (단조 증가해야 건강) | 우상향 |
| **MFE/MAE** | 신호 후 최대상승/최대하락 (손익비) | MFE/MAE ≥ 1.5 |
| **커버리지** | 하루 평균 S/A 신호 개수 | 0이면 무용, 너무 많아도 X |

### 5.4 워크포워드 (과최적화 방지)
- 기간을 In-sample(파라미터 튜닝) / Out-of-sample(검증)로 분리.
- 예: 2019~2023 튜닝 → 2024~2025 검증. OOS에서 lift가 무너지면 과최적화로 판정.
- 팩터 가중치는 OOS 성적으로만 채택.

### 5.5 산출물
```
backtest_report_YYYYMMDD.md  — 지표 표 + 분위 그래프(텍스트) + 팩터별 단독 lift
backtest_trades.parquet      — 모든 (날짜,종목,점수,라벨,MFE,MAE) 원장
```

## 제9부: 구현 로드맵 (단계별 · 각 단계 테스트 통과 후 다음)

| Phase | 산출물 | 완료 기준 |
|---|---|---|
| **P1. 데이터+유니버스** | datafeed.py, universe.py | mock 테스트 통과 + 실제 1일치 전종목 수급 성공 |
| **P2. 팩터+스코어** | factors.py, score.py, indicators 보강 | 합성데이터 단위테스트 100% + 룩어헤드 봉인 |

## 제10부: 알려진 한계 & 리스크 (정직 고지)

1. **생존편향**: 과거 종목 리스트에 상폐 종목이 빠져 있으면 백테스트가 낙관 편향. → 가능하면 상폐 포함 유니버스 사용, 최소한 한계로 명시.
2. **데이터 품질**: 액면분할·유증·합병의 수정주가 처리(FDR 수정주가 사용). 거래정지 구간 갭.
3. **과최적화**: 팩터·임계값이 많을수록 과거에만 맞춰짐. → 팩터 수 최소화, OOS 검증 필수.
4. **라벨 민감도**: K·X를 바꾸면 결론이 흔들림 → 여러 (K,X) 조합에서 견고한지 확인.
5. **체결 현실성**: "신호일 종가 진입"은 이상화. 실제론 슬리피지·갭·상한가 미체결. 발굴 알림이라 1차 영향은 적지만, 향후 자동매매 확장 시 핵심.
6. **레짐 의존**: 강세장에선 다 오르고 약세장에선 다 빠짐 → 시장 레짐 필터(지수 추세) 향후 고려.
